# (RESTFULL-BOOKER) Hotel Website Automation Testing Project

This is an automated testing project developed using **Selenium WebDriver with Java**, **Maven**, **Bonigarcia WebDriverManager**, and **ExtentReports (Aventstack)**. The project automates the testing of key functionalities in a hotel management web application, covering both the **admin panel** and **front-end pages**.

This project was developed as part of a graduation requirement from the **Digital Egypt Pioneers Initiative**, where I completed a full Software Testing learning path (Manual, API, and Automation testing).

## üß™ Test Coverage

The project covers the following four user stories:

### 1. Login Page (Admin Panel)
- **TC01**: Verify admin login with valid credentials.
- **TC02**: Verify login with invalid username/password.
- **TC03**: Attempt login with blank fields ‚Üí verify validation message.
- **TC04**: Validate successful login redirects to admin dashboard.

### 2. Contact Form (Front Page)
- **TC05**: Submit valid contact form (name, email, phone, subject, description) ‚Üí verify success message.
- **TC06**: Submit form with missing fields ‚Üí verify validation errors.
- **TC07**: Submit invalid email format ‚Üí verify error.

### 3. Book a Room
- **TC08**: Select room ‚Üí enter valid booking details ‚Üí submit ‚Üí verify confirmation.
- **TC09**: Attempt booking with overlapping dates ‚Üí check for rejection or proper handling.
- **TC10**: Book without filling required fields ‚Üí verify validation.
- **TC11**: Book with invalid date range (end date before start) ‚Üí verify error.

### 4. Room Management (Admin Panel)
- **TC12**: After login, add a new room (room type, price, accessibility, image) ‚Üí verify room is listed.
- **TC13**: Edit an existing room ‚Üí update info ‚Üí verify changes persist.
- **TC14**: Delete a room ‚Üí confirm it no longer appears in the list.
- **TC15**: Add room with missing fields ‚Üí verify validation.
